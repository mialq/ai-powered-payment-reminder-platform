# Tom de Mensagem

## Objetivo

Definir o padrão de linguagem para mensagens de lembrete de pagamento geradas pela solução.

## Características do Tom

As mensagens devem ser:

- profissionais
- preventivas
- objetivas
- respeitosas
- fáceis de entender

## O que a Mensagem Deve Fazer

- informar o vencimento próximo
- incentivar a regularização dentro do prazo
- manter tom neutro e não agressivo
- apoiar uma comunicação preventiva e operacional

## O que a Mensagem Não Deve Fazer

- ameaçar o cliente
- usar linguagem de cobrança pesada antes do vencimento
- inventar dados não disponíveis
- usar placeholders genéricos sem contexto real
- gerar texto automático para clientes bloqueados

## Regras de Redação

- priorizar frases curtas
- usar português claro
- evitar excesso de formalidade
- evitar termos jurídicos ou intimidadoras
- incluir, quando adequado, a orientação para desconsiderar a mensagem caso o pagamento já tenha sido efetuado

## Casos Bloqueados

Quando o cliente estiver bloqueado:

- não gerar mensagem de lembrete de cobrança
- gerar apenas indicação operacional de revisão manual ou cadastral, se necessário

## Resultado Esperado

A mensagem final deve refletir o objetivo da comunicação preventiva: lembrar, orientar e apoiar a regularização, sem gerar atrito desnecessário com o cliente.
